# 旅游手册生成器 · 设计规格（Design Spec）

> 创建日期：2026-04-17
> 状态：已定稿（经用户确认）
> UI 布局：方案 B（视觉预览型）

---

## 1. 产品定位

**一句话描述：** 一款浏览器端旅行手册 PPT 生成工具，支持手动填写和 AI 智能解析两种方式输入行程数据，预览 5 套主题配色，一键导出 .pptx 文件。

**目标用户：** 旅行社从业者、旅游博主、户外俱乐部领队、活动策划人员
**核心场景：** 快速生成专业级旅行手册，用于客户沟通、团队分发、报名宣传

**左侧边栏：** 72px 宽，图标 + 文字，仅作导航跳转，当前 Tab 高亮
**中间预览区：** 220px 宽，显示当天 PPT 效果的实时缩略图，下方 D1-D7 天数切换
**右侧内容区：** flex:1，表单填写区，支持滚动

### 2.2 为什么选择方案 B

方案 B 的核心优势是**实时预览**，用户在填写表单时能即时看到当天行程的 PPT 效果。旅行手册是视觉型产品，这个反馈回路是其他方案无法提供的。方案 C（步骤引导型）的优点是对新手友好，但限制了同时编辑多个章节的能力——对于需要批量处理行程的旅行社用户来说，这是关键缺陷。

### 3.2 字体与间距

- 主字体：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif`
- 表单字号：12px，标签字号：11px，字重 600
- 圆角：卡片 12px，按钮 8px，输入框 6px
- 阴影：`0 4px 20px rgba(0,0,0,0.08)`（表单卡片）
- 间距基准：8px 网格

### 3.3 加载状态

首次加载 PptxGenJS 库时显示全屏遮罩：
- 进度条 + 状态文字（"正在准备..." → "首次下载 PPT 库..." → "库已缓存"）
- 首次下载约需 2-5 秒（取决于网络），之后永久本地缓存

## 4. PPT 页面结构（14 页）

| 页码 | 类型 | 内容 |
|------|------|------|
| 1 | 封面 | 主标题、副标题、装饰线、天数/公里数、城市路线、联系信息 |
| 2 | 行程概览 | 城市节点动态间距图、底部统计数据 |
| 3-N | 每日行程 | 左侧缩略图（图片或纯色底）、右侧：标题/描述/住宿/餐饮/亮点 |
| N+1 | 美食推荐 | 3列卡片网格，图片+名称+描述 |
| N+2 | 温馨提示 | 图标+文字列表 |
| N+3 | 费用说明 | 双栏对比：包含项目 vs 不含项目 |
| N+4 | 联系我们 | 居中联系信息 + 二维码 |
| N+5 | 封底 | 主色背景 + 主标题 + 联系信息 |

**单位换算（16:9）：** 全宽 `w:10`，全高 `h:5.625`，所有坐标使用数字英寸值，不使用百分比字符串。

## 5. 功能规格

### 5.1 表单填写（手动模式）

- 6 个 Tab 页面，支持随时切换
- 数据保存在 JS 内存（`state` 对象），不自动持久化到 localStorage
- 图片上传：触发 `<input type="file">`，自动压缩至 ≤2MB（Canvas API），宽最大 1920px，JPEG 质量逐步降低至满足大小要求

### 5.2 AI 智能解析（DeepSeek）

- 触发：底部"🤖 AI智能解析"按钮 → 弹窗 → 粘贴行程文本
- API：`https://api.deepseek.com/chat/completions`，OpenAI 兼容格式
- 模型：`deepseek-chat`
- 认证：`Authorization: Bearer ${apiKey}`，Key 存于 localStorage
- Prompt：含防御性指令，强制要求返回 JSON
- 错误处理：401（Key无效）/ 429（请求过快）/ 网络错误 / JSON解析失败，各有对应用户文案
- 解析结果直接写入 `state`，完成后自动关闭弹窗并重新渲染

### 5.3 PPT 生成

- 触发：底部"🎯 一键生成 PPT"按钮
- 引擎：PptxGenJS 3.12.0，首次从 CDN 加载并缓存至 localStorage
- 进度提示：按钮旁显示"生成中...第N步"
- 下载：Blob URL + `a.click()`，文件名为主标题（去特殊字符）
- 防重复：生成中禁用按钮

### 5.4 主题切换

- 5 个主题按钮在顶部导航栏，点击即时切换
- `applyTheme()` 更新 CSS 变量（`--color-*`），同步更新 PPT 生成参数
- 记忆偏好：存 `preferred_theme` 至 localStorage

## 7. 技术架构

**三层模块化（无构建工具）：**

```
travel-ppt-generator.html   ← 入口，HTML骨架 + 内联CSS
├── state.js               ← 全局状态 + THEMES配置 + localStorage读写
├── ppt-engine.js          ← PptxGenJS加载器 + 所有buildSlide函数 + handleGenerate
└── ui.js                 ← 所有render函数 + 所有事件绑定（事件委托）
```

**关键架构决策：**
- PptxGenJS：minified 源码从 CDN 首次加载，存入 localStorage（含版本号），零 CDN 依赖
- 图片压缩：Canvas API，迭代降低 JPEG 质量至 ≤2MB
- 事件绑定：全用事件委托，bindGlobalEvents() 只在 DOMContentLoaded 时执行一次
- 主题切换：CSS 变量 + `document.documentElement.style.setProperty()`
- 下载拦截：Blob + setTimeout(revokeObjectURL) + 错误 catch

## 9. 验证方式

1. 在浏览器打开 `travel-ppt-generator.html`
2. 首次加载应看到加载遮罩（2-5秒），之后消失
3. 顶部切换5套主题，观察 CSS 变量实时生效
4. 切换到"每日行程" Tab，点击天数切换，预览区内容变化
5. 填写任意一天行程，点击"🎯 一键生成 PPT"，观察进度提示 → 下载 .pptx
6. 用 PowerPoint/WPS 打开下载的文件，检查封面、概览、D1行程3页内容
